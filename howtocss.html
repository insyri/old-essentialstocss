The CSS Syntax.

CSS Introduction.

The fundamental goal of the Cascading Stylesheet language is to allow a service engine to edit HTML elements, such as coloring, repositioning, and generally artistically revising the HTML. Some important components as the main building blocks of CSS are:

  • Selector
  • Property
  • Value

    • The Selector is the specific line of code in HTML that will be edited with the following properties.

    The Property is the actual identifier, meaning that this is the specific feature that is looked upon.

    The Value describes how it will be handled in the engine, meaning the ammount of x will happen.

    Note the actual CSS code is in the </style> indent, the rest is HTML.

    CSS Declarations.

    CSS is a series of setting different HTML declarations all making up a function. A value and property pair is dubbed ‘a declaration’, this is used to style every tag and element in the HTML document type. A list of CSS Refrence Values from w3schools.com

    Selectors.

    In the beginning of every CSS code, the first string will be the defined object. This can be multiple or a specific part of an HTML object.
  • Selector Tag
  • Property Tag
  • Punctuation Tag
  • Value Tag
    •                 
                    <!DOCTYPE html>
                    <html>
                        <body>
                            <style>
                                html {
                                    background‐color: #E4264D;
                                    font‐family: Arial, Helvetica, sans‐serif;
                                    color: #EBEBEB;
                                }
                            </style>
                            <p>
                                Hello&comma; world&excl;
                            </p>
                        </body>
                    </html>

    Output:

    selectorsexampleoutput1.html

    Notes:

  • Quick facts: HTML code always have < and > symbols to reference an API element, <style> refers to CSS in html, </style> refers to the ending of the CSS code, so the engine can start loading HTML again.
  • <p> and </p> refers to a paragraph setting via HTML API, in other words, this is ideal for writing paragraphs with.
  • The &comma; and &excl; are HTML entities, these are used for special unicode charcters found in the HTML API. Let’s say we want to display HTML code on an HTML embedded site, the engine will get confused and subject the example excerpt to actual HTML code into the site, this avoids that problem with this.
  • The semicolon ; on the last line is unreasonable, the CSS code is not continuing therefore it is optional to leave it blank, however, it’s good practice.
    • Other things can be added to the selector, as shown below. 
                    
                    <!DOCTYPE html>
                    <html>
                        <body>
                            <div class="custom">
                                <style>
                                    .custom, div.h {
                                    background‐color: #E4264D;
                                    font‐family: Arial, Helvetica, sans‐serif;
                                    color: #EBEBEB;
                                    }
                                </style>
                                <p>
                                    Hello&comma; world&excl;
                                </p>
                            </div>
                            <div class="h">
                                Is this thing on&quest;
                            </div>
                            <div class="a">
                                We come from wotffagnraa&comma; do you copy&quest;
                            </div>
                        </body>
                    </html>

  • You can also do .mycustomselector.partone, to specify even more.
    • Simply putting * as a selector, it will apply to ALL elements, it will not override any specified CSS code, however, it will stylize any unspecified elements. Example below: 
                    
                    <!DOCTYPE html>
                    <html>
                        <body>
                            <div class="mycustomselector">
                                <style>
                                    .mycustomselector {
                                        background‐color: #4de426;

                                        color: #EBEBEB;
                                    }
                                    * {
                                        font‐family: Arial, Helvetica, sans‐serif;
                                        color: #264DE4;
                                    }
                                </style>
                                <p>
                                    Hello&comma; world&excl;
                                </p>
                            </div>
                        </body>
                    </html>

                                <div>
                                    <style>
                                        body {
                                            background‐color: #4de426;
                                        }
                                    </style>
                                </div>
                            </body>
                        </html>

    @font‐face

    This special code is an at rule, in this case, specifies what font you want to display in custom styles. More information from the MDN Mozilla Web Docs. But basically, it is formatted like this: 
                        
                        <!DOCTYPE html>
                        <html>
                            <body>
                                <p>
                                    Hello&comma; world&excl;
                                </p>
                                <style>
                                @font‐face {
                                    font‐family: CircularStd‐Medium;
                                    src: url('CircularStd‐Medium.ttf');
                                }
                                    p {
                                        font‐family: CircularStd‐Medium.ttf;
                                    }
                                </style>
                            </body>
                        </html>
